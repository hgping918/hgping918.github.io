JMeter插件和工具

一、JMeter监控内存和CPU的工具—jconsole

  1. jconsole运行环境
    配置好JMeter运行环境，即可使用 jconsole工具。【JMeter下载地址】

  2. jconsole运行步骤
    点击：Windows开始 → 运行 → 输入cmd → 命令提示符中输入jconsole并回车 → 弹出“java监视和管理控制台”。

  3. jconsole本地测试
    在java监视和管理控制台 → 勾选“本地进程” → 点击“sun.tools.jconsole.JConsloe” → 点击“连接”后进入监控界面（见图1）。

    图1 Java监视和管理控制台

  4. jconsole远程测试
    在java监视和管理控制台 → 勾选“远程进程”输入远程服务器的用法（见图2） → 再输入远程服务器的用户名和密码 → 点击“连接”可进入监控界面。

    图2 Jconsole新建连接

    注：若点击“连接”时弹出“安全连接失败，是否以不安全的方式重试？”提示，点击“不安全”即可连接进入监控界面。

　

二、JMeter监控内存和CUP的插件—JMeterPlugins

JMeter对于内存和CPU的监控存在不足，通过系统的资源监控器来观察，一闪而过又不会记录下来，不便于查阅和对比。Google开发了一款专门用于监控的JMeter插件—JMeterPlugins，弥补了JMeter这方面的不足。【以Windows为例介绍JMeter-Plugins】

  • JMeterPlugins的下载
    当前最新版本：JMeterPlugins-Standard-1.3.0.zip(标准版)
    插件下载地址：http://www.jmeter-plugins.org/downloads/all/

  • JMeterPlugins的配置
    解压JMeterPlugins-Standard-1.3.0.zip,仅将解压包lib\ext\目录下的JMeterPlugins-Standard.jar复制到JMeter的bin\ext\目录下；

  • JMeterPlugins的使用
    启动JMeter：双击jmeter.bat → 添加(add) → 线程组、定时器、前后处理器、监听器都相应增添有“jp@pc”开头的文件。
    添加监控内存监听器：添加(add) → 监听器(listener) → “jp@pc - PerfMon Metrics Collector”。
    运行ServerAgent中ServerAgent.bat
    监控内存需要使用监听器中：“jp@gc - PerfMon Metrics Collector”。
